Use Blender as Python module

19 Mar 2015

I have my bpy module work today by following the directions of BlenderAsPyModule. The only difference is in that I do not copy *.dll and bpy.pyd files to site-packages. Instead, I set environment variable PATH pointing to those *.dlls and set sys.path pointing to bpy.pyd. However, a copy of bin/<blender-version> in the Python’s root directory seems inevitable.

Note that the architecture of Python and bpy module should be matched. That is, you cannot imported an x86-64 build of bpy module in an x86 version of Python. Otherwise you would get a “DLL load failed” error.

There is also a minor building error in the revision of Blender I used (ff1ed872d93b5c40f1de599d79084d48f213921d, x86-64 build). I have to add header path for utfconv.h and add definition of WITH_INPUT_IME manually to the project bf_intern_ghost.
